## 前言

当你想要识别一个 IP 是否属于腾讯云，或者在做安全策略时需要放行腾讯云的所有出口地址段，你会怎么做？

去腾讯云控制台翻文档？去 IPIP.net 查 Whois？都不是最优雅的路径。

[pdflibr.com](https://www.pdflibr.com) 维护了一份公开的云服务提供商 IP 列表，其中就包含了腾讯云的 IPv4 与 IPv6 地址段。这份列表基于腾讯云公开 ASN（Autonomous System Number）聚合而来，数据来源透明、可自由查阅。

本文就从这份列表出发，聊聊腾讯云 IP 地址段的地理分布规律。

---

## 腾讯云的 ASN

腾讯云在列表中出现的 ASN 编号主要是 **AS45090**，对应的组织名称通常为 `Tencent Cloud`。

值得注意的是，编号 3 的香港节点使用的是独立 ASN —— **AS132203**，这是腾讯云早年收购的香港电讯运营商的遗留 ASN，其余近 80 条记录均归属 AS45090。

## 从数据中看出的几个规律

### 1. 上海是腾讯云最大的单一城市节点

从编号数量来看，上海的 IP 段出现频率非常高。42.192.0.0/16、1.15.0.0/16、1.116.0.0/15、49.234.0.0/15、81.68.0.0/15、106.54.0.0/16、109.244.32.0/20、109.244.144.0/21、109.244.208.0/20、110.40.128.0/16、111.231.0.0/17 及其多个子网段……几乎覆盖了腾讯云所有主要 IP 段。

这与腾讯云在华东地区建设了大量数据中心有关——上海及周边区域是游戏、视频、电商等业务的高密度部署地。

### 2. "1.x.x.x" 段的战略价值

腾讯云持有大量以 `1.` 开头的公网 IP（1.13、1.14、1.15、1.116 等）。这个段属于早期 APAN/ChinaNet 互联网泡沫时期分配的历史遗留资源，如今由各大云厂商持有。

这些段由于历史久远、在很多防火墙白名单里被默认信任，是云厂商重要的"黄金 IP 段"。

### 3. 42.x.x.x 是腾讯云另一大王牌

42.192.0.0/16、42.193.x.x、42.194.x.x 构成了腾讯云在 42 网段的核心资产。这个段在很多 DDoS 防护场景下会被特别标记。

### 4. 109.244.x.x — 精心切割的多地域子网

109.244.0.0/16 被切分成了 26 个子网（编号 45–70），均匀分配到北京、上海、广州、深圳、成都、重庆、天津、南京 8 个城市，每个城市 2–3 个 /20 或 /21 子网。

这种切割方式非常典型：
- **路由策略灵活**：不同城市的流量可以独立调度
- **故障隔离**：某城市节点出问题不影响其他节点
- **精细化计费**：不同区域的 IP 成本和定价可以独立管理

### 5. 全球化布局：香港 & 台湾

编号 3（香港，AS132203）和编号 19–20（台湾台北）说明腾讯云在亚太地区有直接的 IP 宣告。需要注意的是，香港节点使用了独立 ASN AS132203（腾讯云早年收购的香港电讯资产），与大陆主体 AS45090 不同。

### 6. 复用 / 重叠段的小细节

编号 50（109.244.64.0/24）和编号 51（109.244.64.0/20）其实存在包含关系。这在列表中属于同一大段被不同粒度重复收录的情况，实际使用时应以较细粒度的 /24 为准。

## 有什么用？

了解腾讯云的 IP 段分布，在以下场景非常有用：

- **安全组 / 防火墙规则**：在企业防火墙上放行腾讯云 IP 时，可以参考这份列表
- **爬虫识别**：判断某个 IP 是否来自腾讯云数据中心
- **日志分析**：溯源腾讯云出站的请求归属地域
- **ASN 威胁情报**：将 AS45090 加入威胁情报库，监控相关 IP 段的网络活动
- **DDoS 防护**：腾讯云清洗后的流量会使用特定 IP 段回注，了解这些段有助于区分攻击流量和正常回注

## 数据来源与局限性

需要注意的是：

1. **这份列表只是首批公开条目**（编号 1–80），腾讯云实际持有的 IP 段远不止这些
2. **IP 段会变动**：云厂商会不断购入新段或释放旧段，列表可能有滞后
3. **部分 IP 段可能被二次租售**：即腾讯云的 IP 分配给了客户，客户以自己的名义使用，但 ASN 仍是腾讯云的
4. **香港节点使用独立 ASN**：编号 3 使用 AS132203，识别腾讯云香港节点时需同时关注两个 ASN

如果你需要实时数据，建议直接查询腾讯云官方文档或 RIR（Regional Internet Registry）数据库。
